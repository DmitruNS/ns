Обновленная Схема Финансового Сервиса

graph TD; A[Пользователи] -->|1| B[Backend] C[Банк] -->|2| B D[Хранилище крипты] -->|3| B E[Мобильное приложение] -->|4| B F[Кассы и Магазины] -->|5| B B -->|6| A B -->|7| C B -->|8| D B -->|9| E B -->|10| F

Описание потоков:

  1. 1: Пользователи → Backend: Запросы на операции, авторизация.
  2. 2: Банк → Backend: Переводы денег, авторизация операций.
  3. 3: Хранилище крипты → Backend: Запросы на операции с криптовалютой.
  4. 4: Мобильное приложение → Backend: Запросы на операции, уведомления.
  5. 5: Кассы и Магазины → Backend: Запросы на операции, сверка счетов.
  6. 6: Backend → Пользователи: Ответы на запросы, уведомления.
  7. 7: Backend → Банк: Запросы на переводы, авторизация операций.
  8. 8: Backend → Хранилище крипты: Операции с криптовалютой, запросы.
  9. 9: Backend → Мобильное приложение: Уведомления, ответы на запросы.
  10. 10: Backend → Кассы и Магазины: Сверка счетов, переводы средств.
