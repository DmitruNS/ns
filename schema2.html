Схема проекта по обмену валют

graph TD; A[Пользователь] -->|Поток 1| B[Мобильное Приложение] B -->|Поток 2| C[2FA / Аутентификация] C -->|Поток 3| D[Бэкенд] D -->|Поток 4| E[Хранилище Крипты] D -->|Поток 5| F[Банк / Кассы] F -->|Поток 6| D E -->|Поток 7| D D -->|Поток 8| C C -->|Поток 9| B B -->|Поток 10| A

Описание потоков:

  1. Поток 1: Пользователь ↔️ Мобильное приложение: Интерфейс пользователя.
  2. Поток 2: Мобильное приложение ↔️ 2FA / Аутентификация: Проверки безопасности.
  3. Поток 3: 2FA / Аутентификация ↔️ Бэкенд: Управление данными для аутентификации.
  4. Поток 4: Бэкенд ↔️ Хранилище Крипты: Управление криптовалютами.
  5. Поток 5: Бэкенд ↔️ Банк / Кассы: Управление фиатными валютами.
  6. Поток 6: Банк / Кассы ↔️ Бэкенд: Обратная связь с банком.
  7. Поток 7: Хранилище Крипты ↔️ Бэкенд: Обратная связь с хранилищем.
  8. Поток 8: Бэкенд ↔️ 2FA / Аутентификация: Обратная связь для аутентификации.
  9. Поток 9: 2FA / Аутентификация ↔️ Мобильное приложение: Обратная связь для аутентификации.
  10. Поток 10: Мобильное приложение ↔️ Пользователь: Обратная связь с пользователем.
